# 次回タスク: MNIST CUDA 学習デバッグの再開

## 現在までの進捗 (2026/04/03 達成事項)
1. **CUDA OOM と Capacity Overflow の根本解決:**
   - コンパイラの中核となる `LivenessAnalyzer` と `CodeGenerator` 間の同期バグを修正し、関数の引数 (`self` など) が常に「使用済み(生存期間0)」と判定される問題を解消。
   - `let mut s = self;` などのインプレースなポインタ移動に対して、誤って巨大な `malloc` とディープコピーが走るのを防ぎ、ゼロコストの「参照移動 (Move Semantic)」を確立。
   - SRET（構造体としての戻り値）の受け渡しに `malloc` ではなくスタックの `alloca` を使用するよう最適化。
2. **すべての基本機能テストがパス:**
   - `cargo test` によって各種言語仕様および CUDA デバイスバインディングのテストが完全に動作していることを確認。
3. **ブランチの統合:**
   - 修正内容をベースブランチである `debug-mnist-cuda` にマージし、最新のメモリ安定化パッチを適用完了。
