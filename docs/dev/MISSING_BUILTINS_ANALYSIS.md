# TL言語 汎用言語化に向けた不足分析

> [!NOTE]
> **ステータス更新 (2026-04-08 最新)**:
> - `Vec`, `HashMap`, `Option`, `Result` などのコレクションおよびモナド型のメソッドはすべて実装完了。
> - 数値定数 (`NaN`, `INFINITY`, `MAX`, `MIN`) と判定関数 (`is_nan()`, `is_inf()`) は実装完了。
> - `System::env/platform/command`, `File::append/delete/list_dir/create_dir`, `Path::parent/file_name/extension` は実装完了。
> - `typeof` による `Type` オブジェクト返却は実装完了。
> - `Channel<T>`, `AtomicI64`, `AtomicI32` の並行処理プリミティブは実装完了。
>
> 現在のリストは、時間型・エラー型・トレイト系言語機能にフォーカスしています。

---

## 1. 不足している組み込み型・構造

### 1.1 時間・日付

| 型 | 説明 | 状態 |
|---|---|---|
| `Duration` | 時間間隔の表現（秒・ミリ秒） | ✅ 実装完了 |
| `Instant` | モノトニック時刻（経過時間計測用） | ✅ 実装完了 |
| `Date` / `DateTime` | 日付・時刻の表現 (タイムゾーン考慮) | ✅ 実装完了 |

### 1.2 エラー型

| 型 | 説明 | 状態 |
|---|---|---|
| `Error` トレイト / 基底型 | エラー型の統一インターフェース | ❌ 未実装 |
