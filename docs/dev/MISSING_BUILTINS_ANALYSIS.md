# TL言語 汎用言語化に向けた不足分析

> [!NOTE]
> **ステータス更新 (2026-04-08 最新)**:
> - `Vec`, `HashMap`, `Option`, `Result` などのコレクションおよびモナド型のメソッドはすべて実装完了。
> - 数値定数 (`NaN`, `INFINITY`, `MAX`, `MIN`) と判定関数 (`is_nan()`, `is_inf()`) は実装完了。
> - `System::env/platform/command`, `File::append/delete/list_dir/create_dir`, `Path::parent/file_name/extension` は実装完了。
> - `typeof` による `Type` オブジェクト返却は実装完了。
> - `Channel<T>`, `AtomicI64`, `AtomicI32` の並行処理プリミティブ実装完了。
> - **時間・日付型**: `Duration`, `Instant`, `DateTime` 実装完了。
> - **トレイト（静的・動的ディスパッチ）**: `dyn Trait` によるvtable構築、および演算子オーバーロード(`Add`, `Sub`, `Eq`等)のサポート基盤の実装完了。
> - **🆕 エラー基盤**: `Error` トレイト、および `dyn Error` サポートの実装完了。
> - **🆕 ネットワーク通信**: `net` モジュール (`TcpListener` / `TcpStream`) 実装および FFI 完全安定化完了。
> - **🆕 文字列操作**: `String.to_bytes()` および `String::from_utf8()` によるバイト配列との相互変換の実装完了。
>
> これによりフェーズCのロードマップで定義されていた主要な標準ライブラリ拡充タスクが完了しました。

## 2. 実装完了したフェーズ・計画 (ロードマップ)

これまでの実装により、TL コンパイラは単純なテンソル演算言語から、エコシステムを備えた汎用言語へと基盤が整いました。

### 実装済みのフェーズCタスク

#### 1. 統一エラー基盤 (`Error` トレイト) 【完了済】
- 独自の構造体がエラーを表現できるようになり、`Result<T, E>` の汎用的なエラーハンドリングを強化。
- `dyn Trait` の動的ディスパッチを活用し、`fn source(self) -> Option<dyn Error>` を定義することで、エラーの数珠つなぎとアップキャストを実現。

#### 2. ネットワークモジュール (`net` モジュール) 【完了済】
- 汎用言語として不可欠なTCP/UDPソケット通信ライブラリの基盤開発。
- `TcpStream`, `TcpListener` 等のランタイム実装とラッパー追加を完了。
- FFI 境界での値受け渡しを純粋なプリミティブとポインタに制限することで（`FFI_IMPLEMENTATION_GUIDE.md` 参照）、C ABI とのメモリレイアウト不整合や Segmentation Fault の問題を解決し完全に安定化。
- `String.to_bytes()` および `String::from_utf8(bytes)` メソッドを追加し、ネットワークを飛び交う生バイト配列 (`Vec<u8>`) と文字列表現の双方向変換をサポート完了。
