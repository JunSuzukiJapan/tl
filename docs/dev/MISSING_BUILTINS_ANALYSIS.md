## 2. 今後の実装計画 (ロードマップ)

これまでの実装基盤（フェーズA〜B）の完成および `dyn Trait` の実装完了により、より高度な言語拡張とエコシステム拡張に向けた残存課題へ移行します。

### 次期フェーズの実装計画（フェーズC ~）

#### 1. 統一エラー基盤 (`Error` トレイト)
- **概要**: 独自の構造体がエラーを表現できるようにし、`Result<T, E>` の汎用的なエラーハンドリングを強化します。
- **目標**:
  - `Display` （実装済み）の拡張として `Error` トレイトを定義。
  - Rustに準拠した `fn source(self) -> Option<dyn Error>` のサポートを提供するため、先ほど実装された `dyn Trait` を活用。
  - カスタムエラーと既存の `String` エラーの互換サポート。

#### 2. ネットワークモジュール (`net` モジュール)
- **概要**: 汎用言語として不可欠なTCP/UDPソケット通信ライブラリの基盤開発。
- **目標**:
  - `TcpStream`, `TcpListener` 等の FFI を介したランタイム実装とラッパー追加。
  - **TDD(TEST-DRIVEN DEVELOPMENT)** を用いたイテレーティブな実装。
  - ファイルI/Oモジュール調査結果に基づき、読み取り関数名は `read_to_string` 等のRust互換のインターフェース設計を踏襲する。
