# TL言語 汎用言語化に向けた不足分析

> [!NOTE]
> **ステータス更新 (2026-04 最新)**: `Vec`, `HashMap`, `Option`, `Result` などのコレクションおよびモナド型のメソッドは `codegen/expr.rs` (インラインIR) および `.tl` 内ですべて実装完了しています。
> 現在のリストは、OS連携や真に不足している定数・並行処理機能にフォーカスしています。

## 4. 優先度サマリ（未実装項目）

### 中期ロードマップ
1. **数値クラスのエッジケース対応**: `NaN`, `INFINITY`, `MAX`, `MIN`, 定数と判定関数の追加
2. **システムインターフェース**: `System` / `File` / `Path` などのOSレイヤ機能拡張、コマンド実行など
3. **型システム拡張**: `typeof` での `Type` 構造体返却機構の整備
4. **並行処理**: `Channel<T>`, `Atomic<T>` によるスレッド間通信ベースの開発
5. **言語機能**: トレイト系言語拡張と演算子オーバーロード
