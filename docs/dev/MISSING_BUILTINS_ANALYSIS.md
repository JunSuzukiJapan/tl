# TL言語 汎用言語化に向けた不足分析

> [!NOTE]
> **ステータス更新 (2026-04-08 最新)**:
> - `Vec`, `HashMap`, `Option`, `Result` などのコレクションおよびモナド型のメソッドはすべて実装完了。
> - 数値定数 (`NaN`, `INFINITY`, `MAX`, `MIN`) と判定関数 (`is_nan()`, `is_inf()`) は実装完了。
> - `System::env/platform/command`, `File::append/delete/list_dir/create_dir`, `Path::parent/file_name/extension` は実装完了。
> - `typeof` による `Type` オブジェクト返却は実装完了。
> - `Channel<T>`, `AtomicI64`, `AtomicI32` の並行処理プリミティブ実装完了。
> - **時間・日付型**: `Duration`, `Instant`, `DateTime` 実装完了。
> - **トレイト（静的・動的ディスパッチ）**: `dyn Trait` によるvtable構築、および演算子オーバーロード(`Add`, `Sub`, `Eq`等)のサポート基盤がマージ・安定化しました。
>
> 今後はこれらをベースラインとし、より高度なエコシステム拡張に向けた残存タスク（フェーズC以降）に移行します。

## 2. 今後の実装計画 (ロードマップ)

これまでの実装基盤（フェーズA〜B）の完成および `dyn Trait` のディスパッチ処理の完全安定化により、堅牢な標準ライブラリ開発のための基盤が整いました。

### 次期フェーズの実装計画（フェーズC ~）

#### 1. 統一エラー基盤 (`Error` トレイト) - **CURRENT FOCUS**
- **概要**: 独自の構造体がエラーを表現できるようにし、`Result<T, E>` の汎用的なエラーハンドリングを強化します。
- **目標**:
  - 構造体に対する `Error` トレイトを標準ライブラリで定義。
  - Rustに準拠した `fn source(self) -> Option<dyn Error>` のサポートを提供（実装済みの `dyn Trait` 動的ディスパッチを活用）。
  - カスタムエラーと既存の `String` ベースの簡単なエラーとの互換性・アップキャストサポート。

#### 2. ネットワークモジュール (`net` モジュール)
- **概要**: 汎用言語として不可欠なTCP/UDPソケット通信ライブラリの基盤開発。
- **目標**:
  - `TcpStream`, `TcpListener` 等の FFI を介したランタイム実装とラッパー追加。
  - **TDD(TEST-DRIVEN DEVELOPMENT)** を用いたイテレーティブな実装。
  - 既存の `File` モジュールの設計方針を踏襲し、`read_to_string` 等のRust互換のインターフェース設計を導入。
