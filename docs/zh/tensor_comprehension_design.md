# 张量内涵表达式设计目标

## 张量内涵表达式想要实现的目标

最初构思张量内涵表达式的动机是进行张量缩并。我们希望沿着特定变量对具有多个维度的张量进行缩并。

### 动机：矩阵乘法示例

考虑矩阵乘法 $C_{i,k} = \sum_j A_{i,j} \cdot B_{j,k}$。

传统的循环表示：
```python
for i in range(M):
    for k in range(P):
        for j in range(N):
            C[i, k] += A[i, j] * B[j, k]
```

我们本质上想知道的是：
- **保留的索引**：`i`、`k`（输出张量的维度）
- **缩并的索引**：`j`（求和并消除的维度）

### 张量内涵表达式的构想

因此，我们设计了一种直接描述"保留哪些索引"的方法：

```
[ i, k | { A[i, j] * B[j, k] } ]
```

在这种表示法中：
- `[ i, k |`：输出张量具有 `i` 和 `k` 两个维度
- `A[i, j] * B[j, k]`：张量之间的乘积
- **隐式缩并**：`j` 不出现在输出中，因此自动求和

换句话说，最初的意图是只需"声明要保留的索引"，其余的会自动缩并。

### 与爱因斯坦求和约定的关系

这与爱因斯坦求和约定（Einstein summation convention）的思想相同：

| 操作 | 爱因斯坦记法 | 张量内涵表达式 |
|------|---------------------|------------------|
| 矩阵乘法 | $C_{ik} = A_{ij}B_{jk}$ | `[ i, k \| { A[i,j] * B[j,k] } ]` |
| 迹 | $\text{tr}(A) = A_{ii}$ | `[ \| { A[i,i] } ]`（标量） |
| 外积 | $C_{ij} = a_i b_j$ | `[ i, j \| { a[i] * b[j] } ]` |

### 当前实现

在当前的 TL 语言中，可以**省略生成器**并从**张量形状自动推断**索引范围：

```rust
let A = [i | i <- 0..5 { i }];  // 显式生成器

let C = [i | { A[i] + 1 }];     // 隐式范围推断（从 A 的形状）
```

在 `C` 的示例中，`i` 的范围从张量 `A` 的访问模式自动推断。
这实现了最初"只声明要保留的索引"的简洁表示法理想。

## 来自 Haskell 列表内涵表达式的影响

### 记法的由来

张量内涵表达式的语法受到了 **Haskell 列表内涵表达式**的影响：

```haskell
-- Haskell：列表内涵表达式
[ x * 2 | x <- [1..5] ]          -- [2, 4, 6, 8, 10]
[ (x, y) | x <- [1..3], y <- [1..3], x /= y ]  -- 成对列表
```

```rust
// TL：张量内涵表达式
[ i | i <- 0..5 { i * 2 } ]      // 张量 [0, 2, 4, 6, 8]
```

两者共享 `[ 输出 | 生成器, 过滤器 ]` 的共同结构。

### 生成器的引入

Haskell 的列表内涵表达式具有**生成列表**的能力。
从这个想法出发，我们认为"如果扩展张量内涵表达式，也可以生成张量"，因此实现了**生成器**。

总结来说，张量内涵表达式的发展经历了以下路径：

1. **最初的想法**：现有张量的缩并（`[ i, k | { A[i,j] * B[j,k] } ]`）
2. **Haskell 的影响**：通过生成器生成张量（`[ i | i <- 0..N { expr } ]`）
3. **当前形式**：同时支持两种用例

这使得张量内涵表达式从单纯的缩并记法演变为**统一处理张量生成和变换的表达式**。
