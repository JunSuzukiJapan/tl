# 张量内涵表达式指南

TL 语言提供了一种强大且富有表现力的张量内涵表达式语法，灵感来源于 Haskell 的列表内涵表达式。这允许您简洁地描述张量的创建、逐元素变换和归约。

## 语法概述

基本语法如下：

```rust
[ <indices> | <clauses> { <body> } ]
```

*   **`<indices>`**：表示输出张量维度的变量列表（逗号分隔）。
*   **`<clauses>`**：生成器（范围指定）或条件表达式（过滤器）的列表（逗号分隔）。
*   **`{ <body> }`**：计算每个元素值的块表达式。省略时从上下文推断或默认为 0.0。

---

## 1. 生成器 (Generators)

生成器定义索引变量可以取值的范围。

### 显式范围指定
使用 `<-` 显式指定索引的范围。

```rust
// 创建大小为 5 的一维张量：[0, 1, 2, 3, 4]
let A = [i | i <- 0..5 { i }];
```

### 隐式推断
如果省略了输出 `<indices>` 中使用的变量的生成器，系统会尝试从 `<body>` 中使用的张量形状推断范围。

```rust
// 'i' 的范围从 A 的形状推断
let B = [i | { A[i] * 2 }];
```

---

## 2. 条件表达式（过滤）

通过在 `<clauses>` 部分包含布尔表达式，可以过滤计算。

*   条件为**真 (true)** 时，执行该迭代的计算。
*   条件为**假 (false)** 时，跳过该迭代（值保持 0.0 或不加入归约）。

```rust
// 仅在偶数索引处有值的张量（其余为 0.0）
let Evens = [i | i <- 0..5, (i % 2) == 0 { i }];
// 结果：[0, 0, 2, 0, 4]
```

可以写多个条件，它们被视为逻辑与（AND）。

```rust
let C = [i | i <- 0..10, i > 2, i < 8 { i }];
```

---

## 3. 归约 (Reductions)

在 `<clauses>` 中定义但**不包含**在输出 `<indices>` 中的变量被视为**归约变量**。这些变量会计算总和（Sum）。这是作为**隐式归约分析（爱因斯坦求和约定）**实现的，通过分析体表达式自动检测归约变量。

```rust
// 点积
// 'k' 不在输出 [] 中，因此成为归约变量。
// 'i' 是标量输出（大小 1）的虚拟索引。
let dot = [i | i <- 0..1, k <- 0..N { A[k] * B[k] }];
```

### 条件归约
可以使用条件仅对特定元素求和。

```rust
// 0 到 9 的偶数之和
let SumEvens = [i | i <- 0..1, k <- 0..10, (k % 2) == 0 { k }];
// 结果：[20] (0 + 2 + 4 + 6 + 8)
```

---

## 4. 更多示例

### 矩阵乘法
```rust
// C[i, j] = Sum_k ( A[i, k] * B[k, j] )
// 'k' 对应共享维度并被聚合。
let C = [i, j | i <- 0..M, j <- 0..P, k <- 0..N { A[i, k] * B[k, j] }];
```

### 掩码 / 类 ReLU 操作
```rust
// 正值保留，否则为 0
let ReLU = [i | i <- 0..N, A[i] > 0 { A[i] }];
```

### 对角元素提取
```rust
// 提取行索引等于列索引的元素
let Diag = [i | i <- 0..N, j <- 0..N, i == j { Matrix[i, j] }];
```

---

## 5. 可选体 (Optional Body)

省略**体块** `{ ... }` 时，会生成以下**"智能隐式体"**：

1.  **单个索引：** 索引变量本身作为值使用。
    ```rust
    // A = Tensor[[5], f32]: [0, 1, 2, 3, 4]
    let A = [i | i <- 0..5];
    ```

2.  **多个索引：** 生成由索引变量组成的**向量（坐标）**。这使张量的秩增加一。
    ```rust
    // T = Tensor[[N, M, 2], f32]
    // 在 Shape(N, M) 网格的每个点放置向量 [i, j]。
    let grid = [i, j | i <- 0..N, j <- 0..M];
    
    // 具体示例：
    // [i, j | i <- 0..2, j <- 0..2]
    // 结果：
    // [
    //   [[0, 0], [0, 1]], 
    //   [[1, 0], [1, 1]]
    // ]
    // (Shape(2, 2, 2) 的张量)
    ```

这使得对向量场的运算如 `grid - center` 变得直观。
这种行为与 Haskell 的列表内涵表达式形成对照，后者返回扁平列表 `[(0,0), (0,1)...]`（Shape(N*M)）。TL 优先保持张量的多维结构（拓扑）。
