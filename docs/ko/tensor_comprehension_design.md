# 텐서 내포 표현 설계 목표

## 텐서 내포 표현으로 달성하고자 한 것

텐서 내포 표현을 처음 구상한 동기는 텐서 축약을 수행하고 싶었기 때문입니다. 여러 차원을 가진 텐서를 특정 변수로 축약하고자 했습니다.

### 동기: 행렬 곱셈 예시

행렬 곱셈 $C_{i,k} = \sum_j A_{i,j} \cdot B_{j,k}$를 생각해 봅시다.

핵심적으로 알고 싶은 것:
- **유지할 인덱스**: `i`, `k` (출력 텐서의 차원)
- **축약할 인덱스**: `j` (합산하여 제거되는 차원)

### 텐서 내포 표현의 아이디어

"어떤 인덱스를 유지할지"를 직접 기술하는 방법:

```
[ i, k | { A[i, j] * B[j, k] } ]
```

### 아인슈타인 합 규약과의 관계

이것은 아인슈타인의 합 규약(Einstein summation convention)과 동일한 발상입니다.

### 현재 구현

현재 TL 언어에서는 **생성기를 생략**하고 **텐서 형상에서 자동 추론**할 수 있습니다:

```rust
let A = [i | i <- 0..5 { i }];  // 명시적 생성기
let C = [i | { A[i] + 1 }];     // 암시적 범위 추론 (A의 형상에서)
```

## Haskell 리스트 내포 표현의 영향

텐서 내포 표현의 구문은 **Haskell 리스트 내포 표현**에서 영향을 받았습니다.

발전 경로:
1. **최초 아이디어**: 기존 텐서의 축약 (`[ i, k | { A[i,j] * B[j,k] } ]`)
2. **Haskell 영향**: 생성기를 통한 텐서 생성 (`[ i | i <- 0..N { expr } ]`)
3. **현재 형태**: 두 가지 용도를 모두 지원

이를 통해 텐서 내포 표현은 단순한 축약 표기법에서 **텐서의 생성과 변환을 통합적으로 다루는 표현**으로 발전했습니다.
