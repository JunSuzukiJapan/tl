# 텐서 로직: 단순한 텐서보다 더 많은 문제를 풀 수 있는 이유

## 개요

**텐서 로직**은 텐서 계산과 논리적 추론을 통합한 프로그래밍 패러다임입니다. 전통적인 텐서 프레임워크(PyTorch, TensorFlow 등)가 주로 수치 계산과 미분 가능한 함수에 초점을 맞추는 반면, 텐서 로직은 **논리적 구조**, **제약 충족**, **기호적 추론**을 추가하여 해결 가능한 문제의 범위를 크게 확장합니다.

---

## 1. 전통적 텐서 계산의 한계

- **이산 최적화**: 정수 해와 조합 문제에 약함
- **제약 표현**: "적어도 하나", "정확히 하나" 등의 논리 제약을 자연스럽게 표현 불가
- **기호적 추론**: "A이면 B" 같은 논리적 관계를 직접 인코딩 불가
- **역문제**: 결과에서 원인을 추론하기 어려움

## 2. 텐서 로직의 확장

논리 연산을 텐서로 표현하고, 제약을 손실 함수로 변환하며, 기호적 구조를 인코딩할 수 있습니다.

| 논리 연산 | 텐서 표현 |
|----------|-----------|
| NOT(A) | `1.0 - A` |
| AND(A,B) | `A * B` |
| OR(A,B) | `A + B - A * B` |

## 3. 텐서 로직으로 해결할 수 있는 새로운 문제 클래스

- **조합 최적화**: N-Queens, TSP, 그래프 착색, SAT
- **역문제**: 역 생명 게임, 역 렌더링, 회로 학습
- **구조화 추론**: 지식 그래프 완성, 프로그램 합성, 패턴 매칭

## 4. 텐서 로직 장점 요약

| 측면 | 전통적 텐서 | 텐서 로직 |
|------|-----------|-----------|
| **문제 표현** | 순수 수치적 | 논리 + 수치 |
| **제약** | 정규화 항으로 추가 | 네이티브 지원 |
| **이산 문제** | 근사 어려움 | 연속 완화로 해결 |
| **역문제** | 특수 아키텍처 필요 | 순전파 미분으로 자연스럽게 |
| **기호적 추론** | 미지원 | 텐서 연산으로 변환 |

## 5. 결론

텐서 로직은 **미분 가능 프로그래밍**의 힘과 **기호적 추론**의 표현력을 결합하여 전통적 프레임워크로는 어려웠던 문제 클래스를 해결할 수 있게 합니다.
