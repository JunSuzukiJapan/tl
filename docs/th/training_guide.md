# กลไกการฝึกโมเดลที่รองรับ

TensorLanguage (TL) รองรับการฝึกโมเดลโครงข่ายประสาทเทียม นอกเหนือจากการทำงานของเทนเซอร์อันทรงพลัง เอกสารนี้จะอธิบายขั้นตอนการทำงานตั้งแต่การกำหนดโมเดลไปจนถึงการนำลูปการฝึกไปใช้และการบันทึกโมเดลที่ได้รับการฝึกโดยใช้ TL

## 1. แนวคิดการฝึกอบรมขั้นพื้นฐาน

การฝึกอบรมใน TL ดำเนินการตามขั้นตอนต่อไปนี้:

1. **คำจำกัดความของโมเดล**: ใช้ `struct` เพื่อกำหนดเลเยอร์และโมเดลที่เก็บพารามิเตอร์และสถานะ
2. **ส่งต่อ**: คำนวณเอาต์พุตจากเทนเซอร์อินพุตเพื่อรับคะแนนการทำนายหรือบันทึก
3. **การคำนวณการสูญเสียและการย้อนกลับ**: เรียกใช้ `loss.backward()` ในฟังก์ชันการสูญเสีย (เช่น `cross_entropy`) เพื่อคำนวณการไล่ระดับสีของแต่ละพารามิเตอร์
4. **การเพิ่มประสิทธิภาพ**: เรียกใช้ฟังก์ชันการเพิ่มประสิทธิภาพในตัว (เช่น `adam_step`) ในแต่ละพารามิเตอร์เพื่ออัปเดต และรีเซ็ตการไล่ระดับสีด้วย `Tensor::clear_grads()`

## 2. ความแตกต่างระหว่างเทนเซอร์และกราดเทนเซอร์

TL มีเทนเซอร์คงที่หลักสองประเภทที่ใช้สำหรับการคำนวณและการฝึกอบรมเชิงตัวเลข:

- **`Tensor<T, R>`**: ข้อมูลอาร์เรย์หลายมิติมาตรฐาน มันไม่ได้ติดตามการไล่ระดับสี (ประวัติการคำนวณ) ดังนั้นจึงรวดเร็วและมีประสิทธิภาพในการใช้หน่วยความจำ ส่วนใหญ่จะใช้สำหรับ **การประมวลผลข้อมูลระหว่างการอนุมาน** และการจัดเก็บ **สถานะภายในของเครื่องมือเพิ่มประสิทธิภาพ (เช่น โมเมนตัมและความแปรปรวน)**
- **`GradTensor<T, R>`**: เทนเซอร์ติดตามการไล่ระดับสีสำหรับการฝึก โดยจะบันทึกกระบวนการคำนวณ (สร้างกราฟการคำนวณ) และดำเนินการสร้างความแตกต่างอัตโนมัติเพื่อคำนวณการไล่ระดับสีเมื่อมีการเรียก `backward()` คุณต้องใช้ `GradTensor` สำหรับ **พารามิเตอร์ (น้ำหนัก อคติ ฯลฯ) เสมอเพื่อให้เรียนรู้/อัปเดต** โดยอัลกอริธึมการปรับให้เหมาะสม

## 3. คำจำกัดความและการเริ่มต้น

แต่ละเลเยอร์ของโมเดลถูกกำหนดให้เป็น `struct` ตัวอย่างเช่น เลเยอร์เชิงเส้นที่ได้รับการฝึกด้วย Adam Optimizer จำเป็นต้องคงสถานะโมเมนตัม (`m`, `v`) นอกเหนือจากน้ำหนักและอคติ เรากำหนด `GradTensor` ให้กับพารามิเตอร์การฝึก และ 'Tensor` ให้กับสถานะเครื่องมือเพิ่มประสิทธิภาพ

## 4. การใช้ขั้นตอนการเพิ่มประสิทธิภาพ

เพิ่มฟังก์ชัน "ขั้นตอน" ลงในแต่ละเลเยอร์เพื่อดำเนินการอัลกอริธึมการปรับให้เหมาะสม (เช่น อดัม) และอัปเดตสถานะ โดยทั่วไปวิธีการ TL `step` จะใช้การออกแบบที่ไม่เปลี่ยนรูปแบบซึ่งส่งคืนโครงสร้างใหม่หลังจากการอัพเดต
