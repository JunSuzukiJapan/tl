# Цели проектирования тензорных включений

Исходная мотивация — свёртка тензоров: сокращение определённых измерений путём суммирования по конкретным переменным.

## Пример: Умножение матриц
$C_{i,k} = \sum_j A_{i,j} \cdot B_{j,k}$

```
[ i, k | { A[i, j] * B[j, k] } ]
```

## Связь с соглашением Эйнштейна о суммировании

| Операция | Тензорное включение |
|----------|-------------------|
| Произведение матриц | `[ i, k \| { A[i,j] * B[j,k] } ]` |
| Сред | `[ \| { A[i,i] } ]` |
| Внешнее произведение | `[ i, j \| { a[i] * b[j] } ]` |

## Влияние списковых включений Haskell

Эволюция:
1. **Исходная идея**: Свёртка существующих тензоров
2. **Влияние Haskell**: Генерация тензоров с помощью генераторов
3. **Текущая форма**: Поддержка обоих вариантов использования
